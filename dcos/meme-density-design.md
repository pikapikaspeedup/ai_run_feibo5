# 梗密度与惊喜感设计 v2.6（随机整活全案）

> 目标：把梗从"读文字的环节"（选卡/开场/结算）铺进**战斗的每一秒**；
> 把确定性的内容变成**分层随机**——让玩家每一局都能说出一句"这局居然……"。

## 0. 设计哲学

1. **三个时间尺度的惊喜节奏**
   - 秒级：同事嘴替气泡——战场一直有人说话
   - 波级：连杀高光/多杀演出——每一波割草有高潮
   - 局级：办公室随机事件 + 稀有金色时刻——每一局的"剧本"不同
2. **轻数值、重演出**（延续 v2.5 被动设计原则）：所有整活效果数值影响 ≤15%，演出拉满。
3. **随机分层**：高频低奇（气泡 8s 一条）→ 中频中奇（事件 ~1 分钟一个）→ 低频高奇（金色时刻 1-5%，一局 0-2 次）。
4. **防刷屏三原则**：全局节流（同屏 ≤2 条/秒）、个体冷却（每单位 ≥8s）、优先级抢占（事件横幅 > 高光 > 气泡）。

## 1. 系统一：同事嘴替（气泡）——秒级 · P0

### 机制
- 气泡 = 弱化版飘字：小号灰白字、带半透明底、上浮慢（区别于伤害/系统飘字）。
- 触发：事件钩子 → 按物种（human/ai）抽对应语料池。
- 节流：单位冷却 8s + 全局 1s 内最多 2 条 + 屏幕外不触发。

## 6. 技术要点（实现备忘）

- **气泡管线**：`bubble(u, key)` → 查 `COPY.BUBBLES[key][u.species]` → 过滤该单位已说过 → 节流检查（`u.bubbleCd`、`G.bubbleGate`）→ addFloat 弱化样式（小字号、灰白、慢速）。
- **事件引擎**：`G.officeEventT` 倒计时（45~75s）；`OFFICE_EVENTS` 表 `{id, name, dur, cond?, start, tick?, end?}`；运行中事件挂 `G.officeEvent = {id, t}`；持续效果尽量复用现有字段（oaSlowT/stunT/buffs/curses/拾取半径 mods），一次性效果在 start 里做完。
- **连杀**：`pl.killStreak/pl.streakT`（killUnit 玩家击杀分支维护）；单帧多杀用 `G.frameKills`（同一 `G.t` 时间戳累计）。
- **金色时刻**：各入口点（killUnit/咖啡机/spawnChip/电梯/newGame/拾取）加一次 `Math.random()` 判定 + 演出。
- **防刷屏**：事件横幅走 warn（已有节奏）；气泡全局 gate `G.bubbleGate`（1s 窗口计数）。
