# 牛马大逃杀 · 地图系统重设计

> 目标：把当前"漂亮的地砖 + 随机撒家具"升级为**有博弈、可收刮、有掩体、能破坏、随机生成还耐玩**的地图系统。这份文档只谈设计和数值，不写代码；实现拆到 Phase 里逐步落。

---

## 0 · 现状诊断

### 当前地图（v2.0）
- 一整块 2400×2400 木地板 tile 铺满
- 72 张桌子 + 48 件装饰随机撒（绿植/饮水机/打印机/白板），间距≥110×80
- **桌子只做碰撞防重叠**，子弹/敌人可穿透（[core.js:200](../src/game/core.js#L200), [core.js:269](../src/game/core.js#L269)）
- **装饰纯视觉**，无 hitbox、无交互、不掉落
- 拾取物（chip/tech/item/xp）随机 spawn 在空地上，先到先得

### 问题
1. **走位没意义**：所有位置价值一样，直线跑图就行
2. **掩体缺失**：远程武器无处躲、狙击手没有走位反制
3. **地图无差异**：60 秒后玩家已经知道所有内容长什么样
4. **收刮无策略**：拾取物是天上掉馅饼，不是主动决策
5. **无破坏感**：家具打不了、砸不烂、没有 juicy 反馈
6. **随机 ≠ 有趣**：均匀撒点不产生 emergent tactical spots

---

## 1 · 设计目标

按优先级：

1. **战术走位** — 掩体让直线弹药 vs 追踪 vs 环绕产生玩法差异
2. **主动收刮** — 玩家决定"值不值得去 X 区拿 Y 好东西"
3. **破坏反馈** — 打家具有音效、爆炸、掉落，改变地形
4. **地图记忆点** — 每局有可识别的 POI，玩家会说"我死在打印机房了"
5. **随机不重样** — 100 局能感觉出 chunk 组合、每次 loot 密度不同
6. **服务大逃杀节奏** — 缩圈+试用期+转正+老板战 每阶段地图作用不同

## 2 · 区域体系（Zone System）

把地图分成 **6 种主题 chunk**，每块占 400×400 世界坐标（当前世界 2400×2400 = 6×6 网格 = 36 chunks）。

### 2.1 Chunk 类型

| Chunk 类型 | 权重 | 视觉标志 | 主要玩法 | Loot 密度 | 掩体密度 |
|---|---|---|---|---|---|
| **工位森林** (open cubicles) | 30% | 5-8 排桌子成阵 | 标准割草 + 大量掩体 | 中 | 高 |
| **会议室** (meeting room) | 15% | 玻璃隔断+大桌 | 内部狭窄，一门进出，chokepoint | 高 | 极高 |
| **茶水间** (break room) | 12% | 咖啡机+饮水机+微波炉+冰箱 | 治疗+护盾 pickup 密集 | 高(治疗) | 低 |
| **打印复印室** (print room) | 10% | 3-5 台打印机 + 纸堆 | 可破坏道具超密集，tech 芯片富矿 | 极高 | 中 |
| **老板办公室** (boss office) | 8% | 大办公桌+沙发+盆栽 | 稀有装备，但 elite spawn 加倍 | 极高(高稀有度) | 中 |
| **走廊/开放区** (corridor) | 25% | 少家具+饮水机 | 移动通道，几乎无 loot | 低 | 低 |

**填充规则**：
- 中心 1×1 chunk 强制是"老板办公室"（终局 boss 在此现身）
- 4 个角强制"茶水间"（提供 rescue heal，防开局暴毙）
- 剩下 31 chunk 按权重随机
- 相邻 chunk 之间强制留 40px 通道（防被墙堵死）

### 2.2 Zone POI 显示

小地图上不同 chunk 用**不同色块**显示：
- 工位森林：深棕 `#4a3a2a`
- 会议室：蓝色 `#2a4a6a`
- 茶水间：绿色 `#3a6a3a`
- 打印室：橙色 `#6a4a2a`
- 老板办公室：金色 `#7a5a1a`
- 走廊：灰色 `#3a3a3a`

玩家看小地图就能主动规划路线："我先扫打印室拿 tech，再去茶水间回血，最后冲老板办公室"。

## 3 · 地形要素（Terrain Elements）

### 3.1 掩体三级（Cover Tiers）

| Tier | 效果 | HP | 例子 |
|---|---|---|---|
| **T1 全掩体** | 100% 阻挡直线弹药 + 阻挡索敌视线 | 40-80 | 桌子、文件柜、白板、微波炉、复印机、玻璃隔断 |
| **T2 半掩体** | 50% 概率挡弹 + 不挡视线 | 20-40 | 办公椅、饮水机、垃圾桶、打印机 |
| **T3 隐蔽** | 完全不挡弹 + **玩家进入后隐身 1.5s**（bot 追杀丢失目标）| 20 | 高绿植、衣帽架、屏风 |

**关键机制**：
- **直线弹药**（mg/single/sniper/twin/gemini/chatgpt/grok）走碰撞检测，被 T1 完全吸收
- **追踪/爆炸/闪电/激光**（doubao/wenxin/glm/claude）无视 T1（因为绕过 / 面积作用）
- **玩家自身移动**不被 T1 挡（撞飞过去，只有子弹被挡）
- **T3 隐蔽**改变 bot AI 决策：`target = null` 直到玩家开火或再次被看见

### 3.2 可破坏物（Destructibles）

**每个物品打爆后的掉落**（RNG，不是保证）：

| 物品 | HP | 打爆掉落 | 特殊效果 |
|---|---|---|---|
| **文件柜** (T1) | 60 | 60% 掉 tech 芯片 | Pro 品级概率 +20% |
| **打印机** (T2) | 30 | 40% 掉 tech + 20% 掉 chip | 掉 chip 时随机武器 ID |
| **咖啡机** (T2) | 25 | 100% 触发 20 半径回血光环 6s | 治疗每秒 3 |
| **饮水机** (T2) | 25 | 100% 触发 60 半径减速地面 8s | 敌人减速 40% |
| **微波炉** (T1) | 40 | 100% 触发 60 半径爆炸 30 伤 | 有蘑菇云特效 |
| **白板** (T1) | 35 | 30% 掉 xp 大丸子 | 随机技能卡 5% 概率 |
| **消防喷淋** (T2) | 20 | 100% 触发全屏火警：**所有敌人 3s 内每秒 -2 hp** | 整体减速 20% |
| **PPT 展板** (T1) | 50 | 击碎变形为 "画大饼" AoE 15 伤 | |
| **绿植/花瓶** (T3) | 15 | 30% 掉小 xp / 10% 掉小血包 | 无 |
| **座机** (T2) | 20 | 40% 触发"骚扰电话"：全场敌人3s 减 20% 攻速 | 铃声 SFX |
| **垃圾桶** (T2) | 15 | 50% 掉 tech 芯片 | 但是有 15% 概率是"过期"减速 debuff |

**打爆判定**：
- 只有 boss 弹药 / player 弹药 / 主动技能可以打
- **同事/试用期杂鱼不能破坏物品**（防止 AI 秒破坏地图）
- 破坏后 tile 变成"废墟"贴图，仍占位但无掩体效果，5 秒后彻底消失

### 3.3 环境交互（Interactive Props）

不需要打爆，走过或按键触发：

| 物品 | 触发方式 | 效果 |
|---|---|---|
| **咖啡机（未破坏）** | 走近 3 秒 | 每次冲杯咖啡回 15 hp（CD 30 秒，一局最多用 3 次）|
| **打印机（未破坏）** | 按 F | 打一张"离职证明"，30% 概率召唤 HR 精英来袭 |
| **消防栓** | 按 F | 释放减速泡沫，全场敌人 5s 减速 50%（CD 60 秒）|
| **电梯** | 走进 | 每 45 秒开门一次，进入后传送到随机另一台电梯位置 |
| **饮水机（未破坏）** | 按 F | 补充水杯 debuff 免疫 3s |
| **公告板** | 按 F | 随机显示一条 tip：`附近有 X` / `敌人 Y 秒后到` / `老板发怒了` |

### 3.4 隐藏内容（Hidden Loot）

**藏在家具里/后面**：

- **抽屉搜刮**：走到桌子边按 F，30% 掉一张 tech 芯片，70% 掉一条无用文案（幽默）。**每张桌子一次**。
- **打印机卡纸**：50% 概率的打印机有 30% 概率藏着一张"卡住的"tech 卡，需要打 3 拳（贴身键盘攻击）才能拿出。
- **文件柜暗格**：5% 的文件柜是"上锁"的（视觉差别：抽屉半开着），破坏后必掉一个大礼物（distill 蒸馏池随机技能 / 装备 chip）。
- **老板保险柜**：老板办公室 chunk 内有 1 个保险柜，HP 200，破坏后掉一个**紫色武器 chip**。

### 3.5 环境事件（Ambient Hazards）

**周期性触发的地图事件**：

| 事件 | 触发条件 | 效果 |
|---|---|---|
| **消防警报** | 每 90-120 秒 | 全场喷淋 8 秒：所有单位每秒 -1 hp、移速 -20% |
| **停电 3 秒** | 每 150-200 秒 | 屏幕变暗，视线半径缩小到 100px，导航失效 |
| **加班通知** | 每 180-240 秒 | 所有 bot 集火最靠近老板的目标 6 秒 |
| **HR 巡视** | zone.phase >= 2 | 一队 HR 从边缘走进来清场 |
| **快递到货** | 一局 3-5 次 | 从空中掉落一箱物资（chip/tech/heal 混装），落地砸中会伤害 |

## 4 · 随机生成算法（Procgen）

### 4.1 三层生成

```
Layer 1 · Zone Grid（chunk 布局）
    ↓
Layer 2 · Template Fill（每个 chunk 选一个 template）
    ↓
Layer 3 · Micro Randomization（template 内 40% 位置抖动 + 变体替换）
```

### 4.2 Layer 1 · Chunk 布局

```javascript
// 6×6 chunk grid
const gridSize = 6;
const chunkTypes = [
  'cubicles', 'meeting', 'break', 'print', 'boss', 'corridor'
];

// 强制填充
grid[2][2] = 'boss';                 // 中心
grid[0][0] = grid[0][5] = 'break';   // 四角茶水间
grid[5][0] = grid[5][5] = 'break';

// 其余按权重随机 + 邻接约束：
//   - 打印室不能连 3 块（避免 loot 集中）
//   - 走廊必须至少连接一个非走廊 chunk
//   - 每种 chunk 类型至少出现 2 次

for (rest of grid) {
  pickWeighted(chunkTypes, weights, constraints);
}
```

### 4.3 Layer 2 · Template

每种 chunk 类型准备 **3-5 个 template 变体**，避免"每次都长一样"。

**Template 例子 · 会议室 A**：

```
┌────────────────────────────┐    walls: [(0,0)-(400,0), (0,0)-(0,400), (400,0)-(400,400), (100,400)-(400,400)]  // 门在左下
│                            │
│  ┌──────────────┐          │    obstacles:
│  │ 会议长桌      │          │      desk_long: (50,150)-(300,220)
│  └──────────────┘          │      chair: (60,150), (100,150), ..., (300,150), (60,220)...
│  * *  * *  * *             │      whiteboard: (300,50) T1 掩体
│                            │
│  ┌─┐                       │      water_cooler: (50,50)
│  └─┘                       │
└──[door]────────────────────┘
```

Codex 的 imagegen skill 直接给这些 template 出俯视缩略图，方便玩家一眼认路。

### 4.4 Layer 3 · Micro Randomization

在 template 基础上：
- **位置抖动**：每个物品 ±20px 随机偏移
- **变体替换**：`plant` 30% 概率替换为 `coat_rack` 或 `printer`
- **额外掉落**：每个 chunk 额外撒 0-2 个"意外"物品（比如工位区突然出现一个咖啡机）
- **旋转**：桌子 25% 概率横竖翻转

## 5 · 博弈机制（Strategic Interactions）

### 5.1 掩体 vs 武器 counter 关系

| 武器 kind | 面对 T1 掩体 | 面对 T2 掩体 | 面对 T3 隐蔽 |
|---|---|---|---|
| mg / single / sniper / twin / grok | ❌ 完全挡 | ⚠️ 50% 挡 | ✅ 无影响 |
| shotgun | ⚠️ 挡 60% | ✅ 穿透 | ✅ 无影响 |
| lob (画饼) | ✅ **越过掩体**（抛物线） | ✅ 无影响 | ✅ 无影响 |
| homing (追踪) | ⚠️ 绕过一次 | ✅ 无影响 | ✅ 无影响 |
| chain (闪电) | ✅ 无影响（面 AOE） | ✅ 无影响 | ✅ 无影响 |
| charge (激光) | ⚠️ **消耗掩体 HP** 而不是直接挡 | ✅ 穿透 | ✅ 无影响 |
| aura (光环) | ✅ 无影响（贴身） | ✅ 无影响 | ✅ 无影响 |

这套 counter 让 **build 选择产生地图偏好**：
- 你玩狙击手 → 主动 kite 到工位森林（多 T1 掩体保命）
- 你玩画饼哥 → 反而喜欢开放区（弧线打远处敌人）
- 你玩追踪流 → 掩体无所谓，专心 kite

### 5.2 破坏博弈

- 破坏一个咖啡机 = 换 15 hp 恢复。**tradeoff**：破坏后此 chunk 少 1 个持续 heal 站
- 破坏一个白板 = 5% 概率技能升级。**tradeoff**：白板可以打 5 拳没爆，你要不要花时间赌？
- 老板保险柜必掉紫色武器 chip，但攻击时**触发 5 秒 boss 传送到你身边**

### 5.3 收刮博弈

- 每张桌子的抽屉每局只能搜刮 1 次，但你要停下 F 键 1 秒——**风险 vs 回报**
- 打印室 loot 密度爆炸，但也是**HR 巡视的必经之路**（AI 优先扫这里）
- 老板办公室有传说 chip，但打开门就会**惊动 Elite T2 精英 3 只**

### 5.4 缩圈的地图叙事

**改造 zone.shrink** 为 **"部门裁员" 叙事**：

- Zone.phase 0 → 1：茶水间关门（4 个角）
- Zone.phase 1 → 2：打印室关门
- Zone.phase 2 → 3：会议室关门
- Zone.phase 3 → 终局：所有 chunk 收缩到"老板办公室"决战

关门 = "本部门被优化了" 红字通告 + 该 chunk 掉在圈外每秒扣血。

## 6 · 收刮内容（Loot Design）

### 6.1 稀有度 vs 位置

| 稀有度 | 位置 | 概率 |
|---|---|---|
| 白 · 普通 | 走廊 / 工位森林 | 到处都是 xp 大丸子 |
| 蓝 · 稀有 | 打印室 / 茶水间 | tech 芯片、副武器 |
| 紫 · 史诗 | 会议室 / 打印室 | 高等级 tech、蒸馏池入口 |
| 金 · 传说 | 老板办公室 / 保险柜 | 传说武器 chip、觉醒配方素材 |
| 彩虹 · 神话 | 需要满足条件解锁的隐藏区 | 一局限 1 个 |

### 6.2 隐藏区解锁

- **打足够多的破坏**：一局累计破坏 30 件物品 → 触发"清洁工"事件，随机 chunk 变成宝藏房
- **触发 5 个环境事件**：所有环境事件都至少触发一次 → 老板保险柜自动开锁
- **专门探索**：走过所有 6 种 chunk 类型 → 电梯开出隐藏楼层

### 6.3 大逃杀阶段梯度

| 阶段 | 主要 loot | 说明 |
|---|---|---|
| 试用期（0-180s）| xp + 少量 tech | 建 build |
| 转正日（180-260s）| chip 井喷（同事空降落地时）| 抢武器 |
| 缩圈期（260-450s）| 破坏物品掉落 + hidden loot | 集中资源 |
| 老板战（450+）| 保险柜、隐藏区 | 决赛专属 |

## 7 · 系统平衡与调优

### 7.1 掩体密度上限

太密 → 玩家永远不吃伤害。太稀 → 掩体系统无意义。

**目标**：任何位置 200px 半径内至少有 1 件 T1 掩体（除了会议室/老板办公室内部）。

### 7.2 破坏物 loot 概率

**期望总掉落**：一局全场破坏 = ~50 件 → 掉落 ~15-20 件真有用的（含 tech/chip/heal）。这样破坏才有价值，也不至于爆炸。

### 7.3 时间成本

- 抽屉搜刮 1 秒 → 敌人接近约 130px（够躲）
- 保险柜 5 秒攻击 → 你要顶着 boss 传送
- 环境事件 CD 90-240s → 一局遇上 3-6 次

### 7.4 存量 vs 消耗

**问题**：破坏物品是有限的（一局最多 200 件），玩家学会后每局都清光 → 疲劳。

**对策**：
- 环境事件"清洁工"重生部分破坏物（免费的持续内容源）
- HR 精英会**主动摆放垃圾**在玩家路径（新的可破坏源）
- 老板战阶段场景重置，所有物品刷新一次（象征"重启一天"）

## 8 · 实现路径（Phased Rollout）

### Phase A · 掩体系统（最小可玩验证）
1. 给现有 obstacles 加 hitbox 阻挡直线弹药
2. 给 desk / cabinet 加 HP 属性
3. render 层显示破损状态（未破 / 打过一次 / 快碎 / 已碎）
4. 目标：**在现有地图基础上加 collide**，体验一次"躲桌子"的乐趣

### Phase B · 破坏与掉落
5. 打爆 obstacle 触发 loot 表
6. SFX + 粒子特效
7. 已破坏 tile 5 秒后消失

### Phase C · Chunk 系统与生成算法
8. 定义 6 种 chunk template
9. Zone Grid 生成
10. Micro randomization

### Phase D · 交互与隐藏
11. 抽屉搜刮
12. 咖啡机 / 打印机 / 消防栓 F 键交互
13. 保险柜 + 隐藏区

### Phase E · 环境事件
14. 消防警报 / 停电 / 加班通知 / HR 巡视 / 快递到货
15. 事件 UI 提示

### Phase F · 缩圈叙事重写
16. 部门裁员式关门
17. 关门 tile 每秒扣血

### Phase G · 美术素材接入
18. 用 `pixel-animation-grid` Skill 生成缺失家具（座机/衣帽架/文件柜/保险柜等）
19. 破损/半破/已破 3 状态贴图（可以一次网格 gen 3 帧）
20. 环境事件视觉（喷淋、警灯、快递箱）

### 优先级建议
**A → B → G(部分素材) → C → D → E → F**

先做 A+B 让玩家立刻感受到"地图有交互"，然后素材跟上，再慢慢做 chunk 系统。F 缩圈叙事最后做（改动最大且和现有系统耦合深）。

## 9 · 数值 / 数据结构 spec

### 9.1 chunk template 结构

```javascript
export const CHUNK_TEMPLATES = {
  cubicles_A: {
```

## 10 · 决策（已定）

1. **掩体 HP**：常量不 scale（roguelike 最佳实践，让掩体后期依然有意义）
2. **同事 AI 用掩体**：是。加"寻找最近 T1 障碍位"到 bot 决策。**部分分型（老油条/摸鱼/关系户）会主动躲**，卷王直冲不用掩体，救火队长有 30% 概率躲
3. **破坏物再生**：**混合设计**——每种 obstacle def 带 `regenerable: true/false` 字段
   - **可再生**：桌子/椅子/绿植（背景重生 30s CD）→ 保持地图活力
   - **不可再生**：文件柜/保险柜/微波炉（一次性稀有）→ 破坏后永远消失
4. **搜刮**：走过即搜（无需按键）——降低认知负担，与 Vampire Survivors 相性一致
5. **地图放大到 3000×3000** → 5×6 = 30 chunk 或 6×5 = 30 chunk 或 6×6 = 36 chunk（400×400 一块，中间留 100px 走廊）
   - 屏幕视野保持 1280×720 视口，玩家看不到全图但小地图能看
6. **老板办公室稀有 loot**：**分层解锁**
   - 试用期：外围可进，只能拿 tech
   - 转正后：中层解锁，可拿蓝紫 chip
   - 老板战触发：保险柜自动开锁，隐藏区可搜刮

## 素材生产策略（补充）
使用 `~/.codex/skills/pixel-animation-grid/SKILL.md` 生成**同物品的多破损状态**：
- 每件破坏物用 2×2 = 4 帧网格：状态 0 完好 / 状态 1 轻微损伤 / 状态 2 严重损伤 / 状态 3 完全破碎
- 4 帧一致性由网格生成保证，切开就是完整破坏动画
- 每件 ~30k tokens 出 4 状态（对比单独 gen 4 次 ~200k）
